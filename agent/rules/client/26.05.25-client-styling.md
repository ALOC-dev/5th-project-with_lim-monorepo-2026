## Client Styling 규칙

이 규칙은 `agent/rules` 하위에서 `client` 영역의 UI 스타일링에 적용한다.

## 1) 기본 원칙

- 기본 import는 `@emotion/styled`를 사용한다.
  ```ts
  import styled from "@emotion/styled";
  ```
- 클래스명/인라인 스타일/유틸 클래스 생성은 기본값으로 금지한다.
- 스타일은 가능하면 하나의 `S` 객체로 통합해 관리한다.
- 루트는 `S.Wrapper`를 기본적으로 둔다.

## 2) 기본 패턴 (권장)

- 기본 생성 형태:
  ```ts
  const S = {
    Wrapper: styled.div`
      ...
    `,
    ...
  };
  ```
- 컴포넌트 사용 예시:
  ```ts
  export const SomeComponent = () => {
    return (
      <S.Wrapper>
        ...
      </S.Wrapper>
    );
  };
  ```
- `Wrapper`는 페이지/섹션/카드 루트 컨테이너의 기본 스타일 포인트로 사용한다.

## 3) 예외 규칙 (예외 허용)

- 외부 라이브러리 컴포넌트의 required `className`/props 주입 패턴이 필요한 경우:
  - 기존 스타일 방식과 충돌이 없고 가독성이 명확하면 예외 허용.
- 이미 존재하는 legacy 파일을 대규모 변경이 불가한 상태에서 수정할 때:
  - 신규 코드에는 위 규칙을 우선 적용하고, 기존 패턴은 점진적으로 이전한다.
- 디자인 시스템 변수/테마 도입이 선결된 범위에서는 해당 설계 규칙을 병행 적용한다.

## 4) 리뷰 체크리스트

- `import styled from "@emotion/styled";`를 사용했는가?
- `S` 객체가 존재하고, 최상위 루트에 `S.Wrapper`를 두었는가?
- 중복 선언/복잡한 인라인 스타일을 피했는가?
- 새 스타일은 타입/컴포넌트 구조와 함께 한 곳에서 관리되는가?

## 5) 적용 기준

- `client` 하위에서 새로 작성/수정하는 UI는 우선 이 규칙을 따른다.
- 기존와 충돌할 때는 본 문서 우선순위를 따르되, 필요 시 문서에 예외 사유를 추가한다.
