# Tools

## UI

- UI 작업 시 DESIGN.md를 적극 참조. 피그마 작업 시에도, 프론트엔드 작업 시에도 마찬가지야.

## Sync 지침

- 어떤 부분에 대해서 sync 맞추라 할 때마다 개발 작업물(코드 읽거나 브라우저 테스트 등), 피그마의 디자인 문서, 노션 내부 문서 이렇게 세 source를 확인해.
- conflict를 해소해. 사용자가 특정 작업물을 우선으로 하라고 했으면 그렇게 하고, 그게 아니라면 어디를 우선으로 할지 conflict 지점마다 사용자에게 질문해. 너무 큰 단위를 block으로 묶지 않도록 주의. 적절히 쪼개서 제시하라는 말이야.

## Figma

- ALOC 어부지리(https://www.figma.com/files/team/1618628187578724701/project/577725895?fuid=1455072441700647159) 내에서만 작업, 그 외의 것들은 절대로 건들지 마.
- 별 말 없으면 "UI" 파일의 하위 페이지에서 작업(https://www.figma.com/design/H5OzPbQABhetO0XyLKAX40/UI?node-id=6-2&p=f&m=draw)
  필요한 경우 페이지를 새로 만들고, 업데이트 요청인 경우 기존 페이지를 수정해.
- 권한 없으면 사용자에게 요청.
- 개발자가 flex를 통한 접근을 선호하니까 참고하고, auto layout을 적극 활용해서 작업. fill, hug, padding, gap, space-between 및 여러 옵션을 사용해. padding이나 gap 없이도 붕 뜨는 공간이 있으면 절대 안된다.
- 페이지와 페이지 간 이동은 화살표로 연결하여 표현해야 해. 대각선으로 가지 말고 수직으로 꺾여야 한다.
